Abstract

Plant disease diagnosis at scale is limited by the lack of datasets that cover enough crops and diseases. We compile a dataset of approximately 1.1M images spanning 53 crops and 259 disease classes, and complement it with a pipeline that automatically generates structured symptom descriptions with per-field provenance, where every fact traces back to a web source with a verbatim supporting quote. We use this combination to build a diagnostic system where each prediction is made by an autonomous reasoning agent that follows a guided chain of thought: it identifies the anatomical context of the test image, narrows candidates using symptom knowledge, and sequentially compares reference images, producing a full reasoning trace at every step. Adding symptom knowledge improves accuracy by an average of 15.2 percentage points at full reference budget, with consistent gains across all three evaluation crops. The system requires no task-specific training, works for any crop with reference images, and will directly benefit from future improvements in foundation model capabilities.

Contributions

  1. Multi-crop disease image dataset. 53 crops and 259 disease classes (~1.1M images), assembled from established benchmarks, expert-curated collections, and community sources, with multi-organ coverage (leaf, stem, root, seed, ear, head).
  2. Source-first disease registry pipeline. Given a crop name, the pipeline automatically produces structured symptom knowledge with per-field provenance. Every fact traces back to a specific web source with a verbatim supporting quote, audited by a domain expert.
  3. Agentic diagnostic pipeline. Each prediction is made by an autonomous reasoning agent that observes the test image, narrows candidates via an anatomical index, consults the symptom KB, and sequentially compares reference images, producing a full human-readable reasoning trace.
  4. Systematic evaluation. Across three crops of varying difficulty (Soybean, Corn, Mango), multiple reference budgets, knowledge base sources, and model tiers (Haiku, Sonnet, Opus).

SAGE Dataset Distribution

Distribution of images across 53 crops and 259 disease classes (~1.1M images). Unlike standard datasets that pair images with labels alone, each disease in SAGE carries structured, source-cited metadata including organ tags, symptom descriptions, source URLs, and verbatim quotes — generated automatically by the registry pipeline.

System Overview

Curation (top): The KB track turns web pages into a source-cited disease KB via automated discovery, extraction, and reconciliation, followed by an expert evidence audit of every cited verbatim quote. The image track collects raw images from multiple sources, applies an expert dedupe of class-name variants, then filters and tags by organ — producing reference/test splits and an anatomical index.
Demonstrated agentic evaluation (bottom): Given a test image, a VLM agent observes the affected organ, narrows candidates via the anatomical index, consults KB symptoms, and sequentially compares reference images in an open-ended loop bounded by reference budget k, producing a prediction with a full step-by-step trace.

Diagnostic Accuracy vs. Reference Budget

Diagnostic accuracy vs. reference budget k across three crops. The agent with internet KB consistently outperforms the no-KB baseline. At k=16, adding the knowledge base lifts Soybean from 31.1% to 51.4% and Corn from 42.0% to 64.8%; on Mango (4 classes) visual comparison alone already exceeds 90%.

Transparent Reasoning Traces

Every prediction comes with a full, human-readable reasoning trace: the agent records which references it chose to view, what it observed at each step, the alternatives it considered, and the final diagnosis with a confidence score. This stands in contrast to single-pass classification, where a model produces a label with no visibility into the decision process.

Cost-Accuracy Tradeoff

Mean accuracy across all three crops with the internet KB. Increasing the reference budget k improves accuracy at growing cost, with diminishing returns beyond k=8. Model quality provides a complementary lever: at k=8, Opus achieves higher mean accuracy than Sonnet at k=16 for comparable cost, while Haiku is the cheapest option but substantially less accurate.
